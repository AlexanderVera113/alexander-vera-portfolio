Aplicación de cifrado de archivos para transferencia segura de datos

Periodo: 3 Noviembre - 14 Noviembre Cliente: Clinica Iquique Rol: Desarrollador de prototipo de seguridad

Aplicación de escritorio para cifrado y descifrado seguro de archivos sensibles utilizando criptografía simétrica.

Contexto del proyecto

Durante mi formación como Técnico en Ciberseguridad quise llevar conceptos teóricos a un entorno práctico. Para ello desarrollé una aplicación de escritorio enfocada en el manejo seguro de archivos sensibles, simulando escenarios de protección de datos médicos.

El proyecto surgió como una iniciativa personal para comprender en profundidad cómo funcionan mecanismos de seguridad como autenticación, cifrado de archivos, almacenamiento seguro de credenciales y registro de eventos.

Posteriormente compartí este proyecto en LinkedIn, lo que generó visibilidad dentro de la comunidad tecnológica y derivó en mi contacto con la empresa NelaTech para participar en proyectos relacionados con transferencia segura de archivos.

Objetivo del proyecto

El objetivo del proyecto fue diseñar una aplicación que permitiera:

  • Cifrar archivos sensibles mediante contraseñas definidas por el usuario
  • Proteger credenciales evitando almacenamiento en texto plano
  • Implementar autenticación básica de usuarios
  • Registrar eventos para mantener trazabilidad de operaciones

Arquitectura del sistema

La aplicación implementa el siguiente flujo de funcionamiento:

  1. Autenticación de usuario: La aplicación cuenta con un sistema de login conectado a una base de datos MySQL que permite validar credenciales de acceso.
  2. Selección de archivo: El usuario selecciona el archivo que desea proteger.
  3. Cifrado: El archivo es cifrado utilizando la librería Cryptography con Fernet, que implementa cifrado simétrico basado en AES y autenticación mediante HMAC.
  4. Gestión segura de claves: Las contraseñas no se almacenan en texto plano. Se generan hashes utilizando hashlib antes de guardarse en la base de datos.
  5. Descifrado: Para recuperar el archivo el usuario debe ingresar la misma contraseña utilizada durante el cifrado, la cual es validada contra el hash almacenado.

Registro de eventos (Logs de seguridad)

La aplicación incorpora un sistema básico de registro de eventos que permite mantener trazabilidad sobre las operaciones realizadas por los usuarios.
Entre los datos registrados se encuentran:

  • nombre del archivo procesado
  • IP del usuario
  • fecha y hora de la operación

Este sistema permitió simular prácticas comunes en sistemas de seguridad donde los logs son utilizados para auditoría y monitoreo.

Funcionalidades principales

CIFRADO DE ARCHIVOS

Permite seleccionar un archivo y cifrarlo utilizando una contraseña definida por el usuario.

DESCIFRADO DE ARCHIVOS

Permite seleccionar un archivo y descifrarlo utilizando una contraseña definida por el usuario.

REGISTRO DE LOGS

Permite registrar y visualizar eventos de seguridad relacionados con las operaciones de cifrado y descifrado. Se muestran IPS, fechas y horas.
